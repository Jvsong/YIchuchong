# 易趣宠（Yiquchong）Web 生态平台 —— 需求分析文档

> 版本：V4.0  
> 日期：2026/06/04  
> 技术栈：Next.js 14 + React 18 + TypeScript + DeepSeek AI  
> 项目定位：宠物智能生态品牌官网与内容平台

---

## 一、项目概述

### 1.1 产品背景

随着城市养宠人群快速增长，宠物主人面临以下核心痛点：

- **走失风险**：宠物外出时无法实时掌握位置，走失后缺乏有效的协寻手段
- **照护知识匮乏**：新手养宠缺乏科学的喂养、运动、健康管理知识
- **服务信任缺失**：寄养、代溜等服务透明度低，主人无法放心托付
- **日常管理碎片化**：喂食、饮水、运动等日常照护缺乏系统化记录与提醒
- **智能设备生态不完善**：市面宠物智能设备各自孤立，缺乏统一的数据联动与管理入口

### 1.2 产品愿景

易趣宠旨在打造以**宠物定位器为核心硬件**，连接**智能设备生态**、**AI养宠建议**、**内容资讯**、**趣味互动**和**宠物服务**的一站式宠物照护平台。通过"硬件+软件+内容+服务"四位一体的模式，让每一次外出、每一次喂养、每一次陪伴都被更好地记录和守护。

### 1.3 目标用户画像

| 用户角色 | 特征描述 | 核心诉求 |
|---------|---------|---------|
| 新手宠主 | 首次养宠，缺乏照护经验 | 获取科学的喂养、运动、健康知识 |
| 城市养狗人群 | 居住在公寓/社区，需要规律遛狗 | 定位安全、路线记录、走失预防 |
| 城市养猫人群 | 室内养猫，关注饮食与健康 | 远程看护、智能喂养、饮水监控 |
| 多宠家庭 | 同时饲养多只宠物 | 设备分组、档案管理、家庭共享 |
| 差旅/加班人群 | 经常短时离家 | 寄养服务、远程投喂、监控看护 |
| 宠物服务门店 | 提供寄养、美容等服务的商家 | 线上展示、客户线索、服务管理 |

### 2.4 智能养宠建议（AI Care）

**需求描述**：利用 AI 大模型（DeepSeek）结合本地规则引擎，为用户生成个性化的每日养宠建议，实现"AI + 养宠"的智能化服务。

**功能点**：

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F-A01 | 养宠参数表单 | 用户选择/输入：宠物类型、品种、年龄、体型、运动量、健康状态、天气、可用时间、定位器佩戴、场景、体重、健康备注、咨询问题 | P0 |
| F-A02 | AI 建议生成 | 调用 DeepSeek API 生成结构化建议（每日建议、运动建议、饮食建议、健康提醒、定位器建议） | P0 |
| F-A03 | 本地规则兜底 | 当 API Key 未配置、超时（12秒）或调用失败时，自动降级到本地规则引擎 | P0 |
| F-A04 | 建议结果展示 | 展示运动时长、次数、强度等量化指标 + 分维度文字建议 | P0 |
| F-A05 | 设备/任务推荐 | 根据宠物档案推荐适配设备和趣味任务 | P1 |
| F-A06 | 表单重置/重新生成 | 支持清空表单和重新生成建议 | P1 |
| F-A07 | 免责声明 | 所有建议标注"仅供参考，不能替代兽医诊断" | P0 |

**本地规则引擎逻辑**：

```
运动时长 = 基础时长(品种×体型) + 年龄调整 + 健康调整 + 天气调整
基础时长参考：
  - 小型犬 35分钟、中型犬 45分钟、中大型犬 55分钟
  - 小型猫 15分钟、中型猫 20分钟、大猫 25分钟
年龄调整：幼年 -8分钟、老年 -15分钟
健康调整：轻微超重 +8分钟、术后恢复 -20分钟、关节敏感 -15分钟
天气调整：炎热 -18分钟、小雨 -10分钟、寒冷 -6分钟
定位器提醒：已佩戴→确认在线+建议开启围栏；未佩戴→建议佩戴
```

**后端接口**：
- `POST /api/pet-advice`：服务端转发至 DeepSeek，前端不直接暴露 API Key
- 超时控制：12秒 AbortController
- 模型：`deepseek-chat`，temperature 0.4，max_tokens 900，返回 JSON

## 四、非功能性需求

### 4.1 性能需求

| 指标 | 目标值 |
|------|--------|
| 首屏加载时间（LCP） | ≤ 2.5秒 |
| 首次输入延迟（FID） | ≤ 100ms |
| 累积布局偏移（CLS） | ≤ 0.1 |
| Lighthouse Performance | ≥ 80 |
| 图片优化 | 使用 next/image，支持懒加载和 WebP |
| API 响应时间 | ≤ 3秒（含 DeepSeek AI 调用） |

### 4.2 兼容性需求

| 平台 | 支持范围 |
|------|---------|
| 桌面浏览器 | Chrome 90+, Safari 14+, Firefox 88+, Edge 90+ |
| 移动浏览器 | iOS Safari 14+, Android Chrome 90+ |
| 屏幕尺寸 | 320px - 2560px（响应式适配） |

### 4.3 安全需求

- 所有 API Key（DeepSeek、数据库等）仅存储在环境变量中，不写入代码或提交 Git
- 生产环境使用 HTTPS
- 管理后台接口必须鉴权
- 密码使用 BCrypt 哈希存储
- 登录接口限流防暴力破解
- 上传文件校验 MIME 类型、大小和内容
- 富文本内容做 XSS 防护
- CORS 白名单限制
- Cookie 设置 HttpOnly、Secure、SameSite

### 4.4 可维护性需求

- 代码按模块组织：pages / components / data / services / types
- 页面通过 service 层获取数据，后期替换数据源只需改 service 层
- TypeScript 严格类型定义
- ESLint 代码规范检查
- 统一的 API 返回格式：`{ code: 0, message: "ok", data: {...} }`
